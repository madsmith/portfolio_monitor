# API v1

Base path: `/api/v1`

---

## GET /api/v1/health

Returns the service health status. No authentication required.

**Response `200 OK`**

```json
{ "status": "ok" }
```

---

## POST /api/v1/login

Authenticates with username and password, returning a bearer token.

**Request body** (`application/json`)

| Field      | Type   | Required | Description       |
|------------|--------|----------|-------------------|
| `username` | string | yes      | Dashboard username |
| `password` | string | yes      | Dashboard password |

**Response `200 OK`**

```json
{ "token": "<auth_key>" }
```

**Response `400 Bad Request`** — malformed or missing JSON body

```json
{ "error": "invalid request body" }
```

**Response `401 Unauthorized`** — wrong credentials

```json
{ "error": "invalid credentials" }
```

The returned `token` is a bearer token to be provided on subsequent authenticated requests:

```
Authorization: Bearer <token>
```

## GET /api/v1/portfolios

Returns a summary of all portfolios. Requires authentication.

**Response `200 OK`**

```json
[
  {
    "id": "a1b2c3d4e5f6a7b8",
    "name": "My Portfolio",
    "total_value": 24000.00,
    "total_cost_basis": 15000.00,
    "total_profit_loss": 9000.00,
    "profit_loss_percentage": 60.0
  }
]
```

| Field                   | Type            | Description                                      |
|-------------------------|-----------------|--------------------------------------------------|
| `id`                    | string          | Portfolio ID (SHA-256 of name, or explicit)      |
| `name`                  | string          | Portfolio name                                   |
| `total_value`           | number \| null  | Current market value of all holdings             |
| `total_cost_basis`      | number \| null  | Total acquisition cost across all assets         |
| `total_profit_loss`     | number \| null  | Unrealised P&L (`total_value - total_cost_basis`) |
| `profit_loss_percentage`| number \| null  | P&L as a percentage of cost basis                |

## GET /api/v1/portfolio/{id}

Returns full details for a single portfolio including per-asset breakdowns. Requires authentication.

**Path parameters**

| Parameter | Description    |
|-----------|----------------|
| `id`      | Portfolio ID   |

**Response `200 OK`**

```json
{
  "id": "a1b2c3d4e5f6a7b8",
  "name": "My Portfolio",
  "total_value": 24000.00,
  "total_cost_basis": 15000.00,
  "total_profit_loss": 9000.00,
  "profit_loss_percentage": 60.0,
  "stocks": [
    {
      "ticker": "AAPL",
      "asset_type": "stock",
      "total_quantity": "150",
      "cost_basis": 15000.00,
      "current_price": 160.00,
      "current_value": 24000.00,
      "profit_loss": 9000.00,
      "profit_loss_percentage": 60.0
    }
  ],
  "currencies": [],
  "crypto": []
}
```

Summary fields are identical to `/api/v1/portfolios`. Each asset in `stocks`, `currencies`, and `crypto` has:

| Field                   | Type            | Description                              |
|-------------------------|-----------------|------------------------------------------|
| `ticker`                | string          | Asset ticker symbol                      |
| `asset_type`            | string          | `"stock"`, `"currency"`, or `"crypto"`   |
| `total_quantity`        | string          | Total quantity held (decimal string)     |
| `cost_basis`            | number \| null  | Total acquisition cost                   |
| `current_price`         | number \| null  | Latest price; `null` if not yet received |
| `current_value`         | number \| null  | `total_quantity × current_price`         |
| `profit_loss`           | number \| null  | Unrealised P&L                           |
| `profit_loss_percentage`| number \| null  | P&L as a percentage of cost basis        |

**Response `404 Not Found`**

If the portfolio could not be located.

```json
{ "error": "not found" }
```
