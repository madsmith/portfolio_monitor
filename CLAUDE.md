## Orientation

Read `SOURCE_MAP.md` at the repo root for a full map of the codebase (backend modules, frontend structure, tests, config, entry points). Do this before exploring files in a new session.

## Coding Standards
Avoid using `any` type in TypeScript. Use specific types instead unless `any` is truly necessary.
Prefer `dict` and `list` over `Dict` and `List` from typing module.
Object oriented programming when possible.
